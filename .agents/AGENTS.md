# Agent Rules

- **Commit Messages**: Always use professional industry-standard commit messages (e.g., Conventional Commits format: `feat:`, `fix:`, `style:`, `refactor:`).
- **Git Command Approval**: When any git command is to be run, you will ask for approval via the terminal command UI (submit button).
- **Explain and Act**: When the user asks 'explain' in their messages, you must first explain and then act. Otherwise, you may proceed directly.
