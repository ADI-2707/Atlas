# Agent Rules

- **Commit Messages**: Always use professional industry-standard commit messages (e.g., Conventional Commits format: `feat:`, `fix:`, `style:`, `refactor:`).
- **Commit Approval**: Never run `git commit` commands automatically without explicit user approval. Always ensure the command is presented to the user via the blue submit button for their review.
- **Explain First & Ask Permission**: Never use file editing tools (like `replace_file_content` or `write_to_file`) or run terminal commands to modify the codebase without first presenting the proposed changes in text and getting explicit user approval.
